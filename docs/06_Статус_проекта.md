# Статус проекта

Документ описывает только технический статус запуска и деплоя. Внутренняя бизнес-логика проекта здесь не раскрывается.

## Структура

```text
backend/
admin-panel/
vercel.json
docs/
```

## Backend

Статус:

- Node.js backend присутствует в папке `backend/`.
- Скрипты запуска описаны в `backend/package.json`.
- Основные команды:

```bash
cd backend
npm ci
npm run dev
```

Порт по умолчанию:

```text
3000
```

Проверка:

```text
http://localhost:3000/api/health
```

## Python/Fox/Camoufox

Статус:

- Дополнительные Python-зависимости описаны в `backend/requirements-fox.txt`.
- Файл содержит:
  - `camoufox>=0.4.11`
  - `playwright>=1.40.0`

Установка:

```bash
cd backend
python3 -m pip install -r requirements-fox.txt
```

Если используется Camoufox, после установки может понадобиться:

```bash
camoufox fetch
```

## Admin-panel

Статус:

- Frontend/admin-panel находится в папке `admin-panel/`.
- Используются React, Vite, TypeScript и Tailwind CSS.
- Скрипты запуска описаны в `admin-panel/package.json`.

Основные команды:

```bash
cd admin-panel
npm ci
npm run dev
```

Локальный адрес обычно:

```text
http://localhost:5173
```

## Vercel

Статус:

- В корне проекта есть `vercel.json`.
- Vercel настроен на сборку `admin-panel`.

Команды из `vercel.json`:

```text
installCommand: cd admin-panel && npm ci
buildCommand: cd admin-panel && npm run build
outputDirectory: admin-panel/dist
```

## Что не настроено автоматически

Перед деплоем и запуском нужно самостоятельно проверить:

- установлены ли Node.js, npm, Python 3 и pip;
- установлены ли npm-зависимости;
- установлен ли Chromium для Playwright;
- нужны ли переменные окружения для конкретного режима запуска;
- доступен ли backend по адресу, который использует frontend.

## Безопасность

В публичный репозиторий нельзя загружать:

- реальные `.env`;
- токены;
- пароли;
- приватные локальные базы данных;
- browser profiles;
- screenshots с приватной информацией.
