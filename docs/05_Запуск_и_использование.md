# Запуск и использование

Документ описывает локальный запуск проекта.

## 1. Установить зависимости

### Backend

```bash
cd backend
npm ci
npm run playwright:install
python3 -m pip install -r requirements-fox.txt
```

Если `npm ci` не работает:

```bash
npm install
```

### Admin-panel

```bash
cd ../admin-panel
npm ci
```

Если `npm ci` не работает:

```bash
npm install
```

## 2. Запустить backend

Откройте первый терминал:

```bash
cd backend
npm run dev
```

Ожидаемый адрес:

```text
http://localhost:3000
```

Проверка API:

```text
http://localhost:3000/api/health
```

Для обычного запуска без watch-режима:

```bash
cd backend
npm start
```

## 3. Запустить admin-panel

Откройте второй терминал:

```bash
cd admin-panel
npm run dev
```

Обычно Vite показывает адрес:

```text
http://localhost:5173
```

Если порт занят, Vite может выбрать другой порт. Используйте адрес из терминала.

## 4. Как frontend видит backend локально

В режиме разработки `admin-panel/vite.config.ts` проксирует запросы `/api` на backend.

По умолчанию:

```text
VITE_API_PROXY_TARGET=http://localhost:3000
```

Если backend запущен на другом порту:

```bash
cd admin-panel
VITE_API_PROXY_TARGET=http://localhost:3001 npm run dev
```

## 5. Сборка admin-panel

```bash
cd admin-panel
npm run build
```

Результат сборки:

```text
admin-panel/dist
```

## 6. Локальный preview сборки

Сначала соберите проект:

```bash
cd admin-panel
npm run build
```

Потом запустите preview:

```bash
npm run preview
```

Важно: backend должен быть запущен отдельно, если admin-panel делает запросы к API.

## 7. Проверка качества frontend

```bash
cd admin-panel
npm run lint
```

## 8. Тесты backend

```bash
cd backend
npm test
```

## 9. Диагностические команды backend

В `backend/package.json` есть дополнительные команды для диагностики Playwright/proxy:

```bash
npm run proxy:diag
npm run proxy:scan-ports
npm run proxy:httpbin-test
npm run proxy:chromium-diagnostic
```

Для этих команд могут понадобиться ENV-переменные proxy. Не вставляйте реальные пароли и токены в документацию или публичный репозиторий.
